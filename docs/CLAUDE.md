# Docs CLAUDE.md

This file provides guidance to Claude Code when working with the Sproot documentation project.

## Project Overview

Sproot Docs is a Docusaurus documentation site that provides API documentation and user guides for the Sproot greenhouse controller system.

**Purpose:** Host technical documentation for the Sproot project including API references and user guides.

**Environment:** Docusaurus (Classic preset) with TypeScript configuration.

## Architecture

**Framework:** Docusaurus (classic preset).

**TypeScript Config:** `docusaurus.config.ts` - Main Docusaurus configuration.

**Build Tool:** Docusaurus builds static HTML from Markdown/MDX content.

## Development Commands

### Build

```bash
# Build the documentation site
npm run build

# Clean build output
npm run clean
```

# Run

### Start local dev server

```bash
npm start
```

### Test

```bash
# Documentation is checked via build
npm run build
```

**Note:** No automated tests - documentation is checked via successful build.
**Note:** No backend service is required for the built site; Docusaurus outputs static HTML, though local development uses a dev server (`npm start`).

## Configuration (`docusaurus.config.ts`)

**Site Configuration:**

```typescript
{
  title: "Sproot Docs",
  tagline: "Documentation for Sproot!",
  baseUrl: "/docs/",
  organizationName: "Ipswich",
  projectName: "Sproot",
}
```

**Presets:**

- Uses `@docusaurus/preset-classic`
- Configures docs routing in `/` (root)
- Sets sidebar path from `./sidebars.ts`

**Theme:**

- Custom CSS from `./src/css/custom.css`
- Prism theme: GitHub (light) / Dracula (dark)
- Color mode: Respects user preference

**Navigation:**

- Navbar with Home, Getting Started (Installation, Setup, Features), How To
- How To includes: Overview, Sensors, Outputs, Automations, Journals, Subcontrollers, Camera, Backups, System Status

**TypeScript:** Shared tsconfig from root.

## Sidebars Configuration (`sidebars.ts`)

**Structure:**

```typescript
const sidebars: SidebarsConfig = {
  gettingStarted: [{ type: "autogenerated", dirName: "getting-started" }],
  howTo: [{ type: "autogenerated", dirName: "how-to" }],
};
```

**Autodoc:** Sidebar automatically generated from folder names.

## Development Tips

## Build Order Requirements

No dependencies on other projects. Can build docs at any time.

**Recommended Build Sequence:**

1. `cd docs && npm run build`

## Docker Deployment

Documentation does not build its own container image. This workspace produces the static site artifact in `docs/build/`:

- Build artifact: `docs/build/`
- Deployment image: `ghcr.io/ipswich/sproot-client:latest` (nginx serves the docs under `/docs`)
- Packaging is handled outside this workspace by the gateway/nginx image, which copies the built docs into `/usr/share/nginx/html/docs`
